# Стиль вывода игрового поля

## Структура

Поле выводится в два блока, разделённых горизонтальной линией:

```
ENEMY BOARD
[Hero] <имя> hp:<хп> [armor: <броня>] [{<атака>/<прочность>}]  mana:<мана>
[<id>] <имя> <атака>/<хп>
[<id>] <имя> <атака>/<хп>
---------------------------------------------------------------
MY BOARD
[Hero] <имя> hp:<хп> [armor: <броня>] [{<атака>/<прочность>}]  mana:<мана>
[<id>] <имя> <атака>/<хп>
```

Сначала идёт сторона противника, затем разделитель, затем своя сторона.

## Герой

```
[Hero] Garrosh-Харрош hp:28 armor: 4 {3/2}  mana:3
```

- `[Hero]` — фиксированный префикс вместо позиции
- `hp:` — текущее здоровье
- `armor: N` — выводится только если броня > 0
- `{атака/прочность}` — выводится только если атака > 0 И это текущий игрок
  - если есть оружие: `{атака/прочность_оружия}`
  - если атака без оружия (например эффект): `{атака}`
- `mana:` — текущая мана

## Существо

```
[3] Wisp 1/1
[7] Murloc Raider 2/3
```

- `[id]` — уникальный числовой ID существа (не позиция на поле)
- затем имя и характеристики `атака/хп`

## Рука (выводится после поля)

```
HAND:
[0] Name: Wisp | Cost: 1
[1] Name: Fireball | Cost: 4
```

- позиция в руке `[0]`, `[1]` и т.д.
- имя и стоимость карты

## Пример полного вывода

```
ENEMY BOARD
[Hero] Garrosh-Харрош hp:30  mana:0
---------------------------------------------------------------
MY BOARD
[Hero] Валира-Валера hp:30  mana:2
[3] Test Minion 1/1
HAND:
[0] Name: Test Minion | Cost: 1
[1] Name: Test Spell | Cost: 3
```

## Ключевые решения

- Существа идентифицируются по **ID**, а не по позиции — это позволяет использовать тот же ID в `TargetSelector` для атаки или нанесения урона
- Герой не имеет числового ID в выводе, обращение к нему через `TargetSelector::oppositeHero` / `TargetSelector::myHero`
- Условные поля (броня, атака) не выводятся если равны нулю — вывод остаётся чистым
